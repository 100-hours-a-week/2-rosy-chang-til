#### 동시성, 스케줄러, 병렬성
|  | 정의 | 비고 |
| --- | --- | --- |
| 동시성 | 하나의 CPU에서 여러개의 작업이 동시에 실행되는 것처럼 보이도록 하는 것 | - 핵심: “하나의 CPU”, “동시에 실행되는 것처럼 보이게” <br> - 마치 1명의 직원이 모든 일을 다하는 것 |
| 병렬성 | 여러개의 CPU 코어를 사용하여 여러 작업을 물리적으로 동시에 처리(실행)함 | - 핵심: “여러 CPU 코어”, “여러 작업을 물리적으로 동시 실행” <br>- 마치 여러명의 직원이 각자 다른 일을 동시에 처리하는 것 |
| 스케쥴러 | 프로세스나 스레드의 작업 실행 순서를 관리하여 시스템 자원을 적절하게 분배함으로써 성능을 최적화하기 위한 역할 | - 핵심: “운영체제의 핵심 구성 요소”, “**시스템 자원 분배”, “실행 순서 관리”** <br>- CPU는 중요하지만 한정적인 자원임으로 여러 프로세스가 나눠 쓸 수 있도록 스케줄러가 관리함. <br>- CPU 이용률을 최대한으로 끌어올리고, 초당 처리할 수 있는 작업량, 총 작업 처리시간, 대기 및 응답 시간을 줄이기 위해 사용함. |

- 동시성과 병렬성의 차이를 설명해주세요

  동시성은 시스템 자원을 절약하기 위한 방법이고, 병렬성은 시간을 절약하면서 생산성을 강화하기 위한 방법

  동시성과 병렬성은 분명히 다른 개념이지만 상호보완적인 관계이다.

    - 멀티 스레드 환경에서의 동시성과 병렬성을 설명해주세요

      동시성은 싱글 코어에서 멀티스레드를 동작시키기 위한 방식으로, **멀티태스킹**을 위해 여러 개의 스레드가 번갈아가면서 실행되는 성질을 말한다. 싱글 코어에서 여러 작업이 동시에 실행되는 것처럼 보이게 하기 위해 짧은 시간 단위로 컨텍스트 스위칭이 일어난다. → 실제로 여러 작업이 동시에 처리되는 것은 아님!

      병렬성은 멀티 코어에서 멀티스레드를 동작시키기 위한 방식으로, 1개 이상의 스레드를 가진 코어들이 동시에 실행되는 성질을 말한다. → 물리적으로 동시에 여러개의 작업을 처리할 수 있다.

      **멀티태스킹**이란?

      퀀텀이라는 아주 짧은 시간동안만 프로세스/스레드가 CPU를 사용하게 함으로써 서로 번갈아가면서 각 프로세스/스레드들이 CPU를 최대한 사용할 수 있게 하는 것 → 각 프로세스/스레드들이 병렬적으로 실행되는 것처럼 보이지만 사실을 아주 짧게 번갈아가면서 실행되는 것!

- 컨텍스트 스위칭이 발생할 때 생기는 부정적인 영향에 대해서 설명해주세요

  컨텍스트 스위칭이 너무 많이 일어나면 성능이 떨어짐. → Why? → 많아질수록 기억해야 하는 작업상태가 늘어나고 비용이 증가 → 컨텍스트 오버헤드 발생

- 스레드풀을 사용하는 이유와 장단점에 대해서 설명해주세요

  스레드를 매번 새롭게 생성, 삭제하여 관리하는 것은 무거운 작업! → 스레드 풀을 사용하여 이미 만들어진 스레드를 재활용함으로써 스레드의 개수를 정하거나 스레드 생성과 소모에 드는 비용(오버헤드)를 줄일 수 있기 때문에 사용함.

  [장점]

    1. 성능 향상 : 스레드의 반복적인 생성과 소모하는 작업을 하지 않고 미리 생성되어있는 스레드를 재활용함으로써 시스템의 성능 향상 가능
    2. 오버헤드 및 비용 감소 : 스레드 풀을 사용하면 수많은 작업을 오버헤드 없이 실행할 수 있고, 스레드 생성과 삭제에 따른 오버헤드와 비용을 줄일 수 있음.

  [단점]

    1. 메모리 낭비 : 한번에 너무 많은 스레드를 생성하면 메모리 낭비가 발생 → 이미 생성된 스레드를 관리하기 위해 메모리를 소비하게 되고, 너무 큰 스레드 풀은 그것 자체로도 오버헤드일 것이고, 자원의 낭비를 초래하게 됨 → 적절한 크기의 스레드풀을 설정해야 함!
- (선택) 프로세스 스케줄링과 스레드 스케줄링의 차이점에 대해서 설명해주세요

  프로세스 스케줄링은 프로세스 간에 CPU 시간을 분배하는 것을 말하고, 스레드 스케줄링은 프로세스 내의 스레드 간에 CPU 시간을 분배하는 것을 말한다.

  프로세스는 독립적인 실행 단위로, 독립적인 메모리 공간을 가지지만, 스레드는 프로세스 내에서 실행되는 단위로 프로세스의 자원과 메모리를 공유하면서 동작한다.
